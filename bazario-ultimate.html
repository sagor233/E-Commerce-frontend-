Core Architecture

Built for scale
& clarity

Clean 3-layer DDD architecture with transactional boundaries, high-precision monetary handling, UUID keys, and full audit trails on every table.

🏗️

DDD Architecture

Separated API, Service, and Persistence layers — each with explicit contracts. Controllers handle HTTP, Services own business logic, Repositories own data.

3-Layer Pattern 01
🔐

Role-Based Auth

Three-tier security model: ROLE_CUSTOMER for shoppers, ROLE_SELLER for vendors, ROLE_ADMIN for full access. BCrypt hashed credentials with account locking.

JWT + BCrypt 02
📦

Inventory Tracking

Real-time stock management with configurable low-stock thresholds. 1:1 product binding via FK UNIQUE constraints. Cascade deletes keep data clean.

Real-Time Stock 03
🗄️

Flyway Migrations

Versioned database schema evolution. V1 establishes core e-commerce tables, V2 adds auth system. Idempotent, auditable, and safe for zero-downtime deployments.

V1 + V2 Migrations 04
💰

Precision Pricing

DECIMAL(19,4) for base_price and discount_price fields — IEEE 754 compliant monetary arithmetic with zero floating-point drift across all financial operations.

DECIMAL(19,4) 05
📖

OpenAPI 3 Docs

Swagger UI auto-generated from SpringDoc annotations. Interactive browser at /swagger-ui.html, spec at /api-docs.

Under the Hood

Enterprise-grade infrastructure

Built on battle-tested Spring Boot 3.3+ with Domain-Driven Design patterns. Every architectural decision optimizes for correctness, maintainability, and developer joy. PostgreSQL with Flyway ensures zero-downtime schema evolution.

Java 21 LTS Spring Boot 3.3+ PostgreSQL 15+ Flyway 9+ Maven 3.9+ OpenAPI 3 Spring Data JPA Spring Security BCrypt Lombok dotenv-java HikariCP
V2__create_auth_tables.sql
-- Enable UUID extension CREATE EXTENSION IF NOT EXISTS "uuid-ossp"; -- 1. Roles Table CREATE TABLE roles ( id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), name VARCHAR(50) NOT NULL UNIQUE, description TEXT, created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP ); -- Pre-populate default roles (idempotent) INSERT INTO roles (name, description) VALUES ('ROLE_CUSTOMER', 'Standard shopping privileges'), ('ROLE_ADMIN', 'Full system access'), ('ROLE_SELLER', 'Manage own products') ON CONFLICT (name) DO NOTHING; -- 2. Users Table CREATE TABLE users ( id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), email VARCHAR(255) NOT NULL UNIQUE, password VARCHAR(255) NOT NULL, first_name VARCHAR(100), last_name VARCHAR(100), phone_number VARCHAR(20) UNIQUE, is_enabled BOOLEAN DEFAULT TRUE, is_account_non_locked BOOLEAN DEFAULT TRUE, is_account_non_expired BOOLEAN DEFAULT TRUE, is_credentials_non_expired BOOLEAN DEFAULT TRUE, created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP ); -- 3. user_roles junction (Many-to-Many) CREATE TABLE user_roles ( user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE, role_id UUID NOT NULL REFERENCES roles(id) ON DELETE CASCADE, granted_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP, PRIMARY KEY (user_id, role_id) ); -- Optimized indexes CREATE INDEX idx_users_email ON users(email); CREATE INDEX idx_users_enabled ON users(is_enabled) WHERE is_enabled = TRUE; CREATE INDEX idx_roles_name ON roles(name);

GET/api/v1/health
System health check. Returns app name, version, environment profile, DB connection status, and server timestamp.
Auth: None · Public
200 OK
GET/api/v1/categories
List all active categories with id, name, slug, description, product count, and timestamps.
Auth: None · Public
200 + Array
POST/api/v1/categories
Create a new product category. Body: {name, slug, description}. Name + slug must be unique.
Auth: ROLE_ADMIN
201 Created
GET/api/v1/products
Paginated product listing. Query params: category, minPrice, maxPrice, isActive, page, size.
Auth: None · Public
200 + Page
POST/api/v1/products
Create a product with name, slug, base_price, discount_price, sku, category_id, and initial stock_quantity.
Auth: ROLE_ADMIN / ROLE_SELLER
201 Created
GET/api/v1/products/{id}
Get product by UUID. Returns full details including inventory status, category, SKU, and both price fields.
Auth: None · Public
200 / 404
PUT/api/v1/products/{id}
Update product details, pricing, category, SKU, or active status by UUID. Full replacement body required.
Auth: ROLE_ADMIN / ROLE_SELLER
200 Updated
DELETE/api/v1/products/{id}
Soft-delete: sets is_active = false. Non-destructive. Inventory record preserved. Admin only.
Auth: ROLE_ADMIN
204 No Content
GET/api/v1/inventory/{productId}
Get real-time stock_quantity and low_stock_threshold. Returns low-stock alert flag if qty ≤ threshold.
Auth: ROLE_ADMIN / ROLE_SELLER
200 / 404
PATCH/api/v1/inventory/{productId}
Update stock_quantity or low_stock_threshold. Partial update — only provided fields modified.
Auth: ROLE_ADMIN / ROLE_SELLER
200 Updated

Register a new customer. Body: {email, password, firstName, lastName, phone}. Password BCrypt-hashed before storage.
Auth: None · Public

Quick Start

Up & running
in minutes

Prerequisites: JDK 21, PostgreSQL 15+, Maven 3.9+. Clone, configure your .env, and Flyway handles the rest.

01
Fork & Clone
Fork the repository to your GitHub account, then clone your fork locally. You'll need JDK 21 and Maven 3.9+ installed.
02
Configure Environment
Copy .env.example to .env and fill in your PostgreSQL credentials, Flyway settings, and application config.
03
Create PostgreSQL Database
Create a database named bazario_dev_db with a user matching your .env credentials.
04
Run Migrations & Start
Run ./mvnw spring-boot:run. Flyway auto-runs V1 + V2 migrations. App starts on port 8080.
05
Explore Swagger UI
Navigate to http://localhost:8080/swagger-ui.html to browse and test all API endpoints interactively.
Terminal · Quick Start
# 1. Clone your fork git clone https://github.com/YOUR_USERNAME/Bazario.git cd Bazario # 2. Configure environment cp .env.example .env # Edit .env with your PostgreSQL credentials # 3. Create the database (psql) psql -U postgres -c "CREATE DATABASE bazario_dev_db;" psql -U postgres -c "CREATE USER dev_user WITH PASSWORD 'dev_password';" psql -U postgres -c "GRANT ALL ON DATABASE bazario_dev_db TO dev_user;" # 4. Start application (Flyway auto-migrates) ./mvnw spring-boot:run # App starts on http://localhost:8080 # Swagger UI: http://localhost:8080/swagger-ui.html # Health check: http://localhost:8080/api/v1/health # 5. Test health endpoint curl http://localhost:8080/api/v1/health # Expected: {"status":"UP","database":{"status":"UP"},...} # 6. Register a user curl -X POST http://localhost:8080/api/v1/auth/register \ -H "Content-Type: application/json" \ -d '{"email":"user@example.com","password":"secret123","firstName":"Jane"}'
